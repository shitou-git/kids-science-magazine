# 少年科普周刊自动生成系统 - 需求文档

## 一、项目概述

### 1.1 项目背景
创建一个自动化程序，每周定期从全网收集适合少年儿童阅读的前沿科普信息，参考"阳光少年报"的内容风格和排版方式，自动生成精美的HTML周刊页面。

### 1.2 项目目标
- **自动化**：无需人工干预，每周自动完成信息采集、编辑、排版全流程
- **高质量**：内容科学准确、语言生动有趣、适合少年儿童阅读
- **美观性**：HTML页面排版精美、图文并茂、阅读体验良好
- **时效性**：紧跟科技前沿，涵盖最新科学发现和技术进展

### 1.3 目标读者
- 6-14岁少年儿童
- 对科学科普感兴趣的小学生和初中生
- 家长和老师可作为课外读物推荐

## 二、功能需求

### 2.1 信息采集模块
| 功能点 | 说明 |
|--------|------|
| 多源信息抓取 | 从多个科普网站、新闻媒体、学术机构网站采集信息 |
| 定时执行 | 每周固定时间自动启动采集任务 |
| 去重过滤 | 自动过滤重复内容、广告内容 |
| 内容分类 | 自动将内容按学科分类（物理、生物、天文、科技、环境等） |

### 2.2 内容编辑模块
| 功能点 | 说明 |
|--------|------|
| 内容筛选 | 根据适龄性、科学性、趣味性筛选优质内容 |
| 语言润色 | 将专业术语转化为少年儿童易懂的语言 |
| 要点提炼 | 提炼核心知识点，突出"为什么重要" |
| 配图匹配 | 为每篇文章匹配合适的配图 |

### 2.3 HTML排版生成模块
| 功能点 | 说明 |
|--------|------|
| 模板化排版 | 使用预设的精美HTML模板 |
| 目录导航 | 生成周刊目录，支持快速跳转 |
| 图文混排 | 文章与配图合理搭配 |
| 响应式设计 | 支持电脑、平板、手机多端阅读 |
| 页码与日期 | 自动标注期数、出版日期 |

### 2.4 管理模块
| 功能点 | 说明 |
|--------|------|
| 历史存档 | 保存每期周刊，支持历史回顾 |
| 内容源管理 | 可配置信息来源网站列表 |
| 任务监控 | 查看采集任务执行状态和日志 |

## 三、内容规划

### 3.1 内容栏目设置（参考阳光少年报风格）

1. **头版头条** - 本周最重要的科学大新闻
2. **前沿科技** - 最新的技术发明和创新
3. **宇宙探索** - 天文航天相关内容
4. **生命科学** - 生物、医学、健康知识
5. **地球家园** - 环境、气候、地理知识
6. **奇趣发现** - 有趣的科学冷知识、新奇发现
7. **科学家故事** - 科学家的励志故事
8. **动手实验** - 简单有趣的家庭小实验

### 3.2 内容风格要求
- **语言风格**：生动活泼、通俗易懂、富有童趣
- **文章长度**：每篇200-500字，适合少年注意力时长
- **知识深度**：深入浅出，不讲复杂公式，重在启发兴趣
- **价值观**：积极向上，培养科学精神和探究意识

## 五、技术实现方案

### 5.1 技术架构

```
定时触发器 → 信息采集器 → 内容处理器 → HTML生成器 → 输出存档
     ↓            ↓            ↓            ↓
   每周X点    多源抓取    筛选/润色/分类   模板渲染
```

### 5.2 技术选型建议

| 模块 | 技术方案 | 说明 |
|------|----------|------|
| 开发语言 | Python | 生态丰富，爬虫和HTML生成都很方便 |
| 网页抓取 | requests + BeautifulSoup / Scrapy | 根据复杂度选择 |
| 内容处理 | 可结合LLM大模型 | 用于内容筛选、语言润色、摘要生成 |
| HTML生成 | Jinja2模板引擎 | 灵活的模板系统 |
| 定时任务 | schedule库 / cron | 实现每周定时执行 |
| 数据存储 | SQLite / JSON文件 | 保存历史记录和配置 |

### 5.3 自动程序运行流程

1. **触发启动**：每周固定时间（如周日早上8点）自动启动
2. **信息采集**：
   - 遍历配置好的信息源网站
   - 抓取最近一周的科普相关文章
   - 提取标题、正文、发布时间、配图等信息
3. **内容处理**：
   - 去重：去除重复或高度相似的文章
   - 筛选：根据关键词、适龄性评分筛选优质内容
   - 分类：自动归类到不同栏目
   - 编辑：润色语言、调整长度、提炼要点
4. **HTML生成**：
   - 选择当期头条
   - 按栏目排版
   - 插入配图
   - 生成目录和导航
   - 渲染最终HTML页面
5. **输出保存**：
   - 保存为独立的HTML文件
   - 记录到期数档案中
   - 可选：发送通知提醒

## 六、HTML页面设计

### 6.1 页面结构

```
┌─────────────────────────────────┐
│        刊头（报头/Logo）         │
│   第X期 · 202X年X月X日 · 周刊    │
├─────────────────────────────────┤
│         目录导航栏               │
├─────────────────────────────────┤
│                                 │
│   ■ 头版头条（大图+大标题）       │
│                                 │
├─────────────────────────────────┤
│   ■ 前沿科技                     │
│   ├─ 文章1（小图+标题+摘要）      │
│   └─ 文章2                       │
├─────────────────────────────────┤
│   ■ 宇宙探索                     │
│   ...                            │
├─────────────────────────────────┤
│   ... 其他栏目 ...               │
├─────────────────────────────────┤
│   页脚：订阅/往期回顾/关于我们    │
└─────────────────────────────────┘
```

### 6.2 设计风格
- **色彩**：明亮活泼，以蓝色、绿色为主色调，富有科技感和生命力
- **字体**：清晰易读，标题醒目，正文行间距舒适
- **配图**：每个栏目至少一张配图，图文并茂
- **互动**：目录可点击跳转，提升阅读体验

## 八、待确认问题

在开始实施前，需要确认以下细节：

1. **每周几、几点发布？** （建议周日或周一早上）
2. **内容来源偏好？** 以中文为主，还是需要翻译英文来源？
3. **是否使用AI大模型？** 用于内容润色、摘要生成等（效果更好但需要API）
4. **HTML页面的风格偏好？** 偏传统报纸风，还是现代杂志风？
5. **是否需要配图自动获取？** 还是使用占位图/固定插图？
6. **每期内容量？** 大概多少篇文章、多少个栏目？
